# payflow-mobile

Android mobile app for the Payflow credit card checkout challenge.

## Stack

- React Native
- TypeScript
- Redux Toolkit
- React Navigation
- Secure/encrypted storage
- Jest
- Android

## Architecture

The app follows a feature-based architecture with Redux/Flux:

- `features/products`
- `features/cart`
- `features/checkout`
- `features/payment`
- `features/transaction-result`
- `shared/api`
- `shared/components`
- `shared/storage`
- `shared/validation`

## Local Setup

Install dependencies:

```bash
npm install
```

Start Metro:

```bash
npm start
```

Run Android:

```bash
npm run android
```

## Tests

Expected commands:

```bash
npm test
npm run test:cov
npm run typecheck
npm run lint
```

Coverage target: more than 80%.

## Android APK

The APK can remain in the default Android build output path generated by React Native/Gradle.

## Environment

Copy `.env.example` to `.env` and fill local values.

Do not commit real credentials or private values.

Current Android emulator default:

```text
MOBILE_API_BASE_URL=http://10.0.2.2:3000
```

The first implementation uses `src/shared/constants/config.ts` for the local API base URL. A runtime env bridge can be added if build variants require it.
