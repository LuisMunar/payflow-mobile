# Contributing

## Branches

Use one branch per feature:

```text
feature/mobile-short-description
```

Open a pull request into `main` when the feature is ready.

## Commit Format

Use small commits with this format:

```text
type(scope): short description
```

Examples:

```text
feat(checkout): add payment summary screen
test(card): cover luhn validation
docs(readme): add android build instructions
```

## Feature Handoff

When a feature is finished, Codex must report:

- branch name;
- summary of changes;
- validation commands executed;
- coverage result when available;
- suggested PR title and description;
- next manual action for the owner to create, review and merge the PR.

Codex does not merge PRs automatically.
